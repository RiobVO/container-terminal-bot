# Бот в группе + мониторинг терминала

## Цель

Превратить бота из персонального инструмента в групповую систему мониторинга контейнерного терминала. 6 операторов работают с ботом в одной приватной группе Telegram. Бот реагирует на команды, транслирует live-ленту действий и автоматически отправляет отчёты по расписанию.

## Компоненты

### 1. Поддержка групповых чатов

**Текущее состояние:** бот жёстко фильтрует `F.chat.type == "private"` в `bot.py:40-41`. В группах молчит.

**Целевое состояние:**
- Бот работает в private-чатах (как раньше) И в разрешённых группах.
- `GROUP_IDS` в `.env` — список разрешённых group chat ID (через запятую). Пустой = только личка.
- Middleware `ChatFilterMiddleware` проверяет: private → пропускаем; group/supergroup → проверяем `chat.id in GROUP_IDS`; остальное → блокируем.
- В группе бот реагирует на команды с @username (`/search@botname`) И без него (если bot privacy mode выключен у BotFather).

**Не меняется:** вся логика хендлеров, FSM, клавиатуры — без изменений. Reply-клавиатуры работают и в группах.

### 2. Live-лента действий

При каждом значимом действии оператора бот отправляет уведомление во все группы из `GROUP_IDS`.

**Триггеры и формат:**

Регистрация контейнера:
```
📥 Новый контейнер
TEMU 6275401 (Maersk) — 40ft
Статус: На терминале
Оператор: @username
```

Вывоз контейнера:
```
🚛 Вывоз
TEMU 6275401 (Maersk) — 40ft
Дней на терминале: 45 | К оплате: $310.00
Оператор: @username
```

Удаление контейнера:
```
🗑 Удалён контейнер
TEMU 6275401 (Maersk)
Оператор: @username
```

**Реализация:**
- `services/group_notify.py` — функция `notify_groups(bot, text)` шлёт сообщение во все группы из конфига.
- Вызывается из хендлеров после успешных мутаций (register, depart, delete).
- Если отправка в группу упала (бот удалён из группы, нет прав) — логируем warning, не блокируем основной флоу.

### 3. Утренний отчёт (06:00)

Автоматическое сообщение в группу каждый день в заданное время.

**Формат:**
```
📊 Утренний отчёт — 12.04.2026

На терминале: 47 контейнеров
В пути: 3 контейнера
Вывезено (вчера): 5

💰 Общая сумма к оплате: $12,450.00

🔴 ТАРИФИКАЦИЯ НАЧАЛАСЬ
├ MSCU 8834210 (Hapag) — 2 дня на тарификации, $40.00
└ OOLU 1234567 (CMA) — 5 дней на тарификации, $100.00

🟡 Скоро тарификация (≤ 3 дня)
├ TEMU 6275401 (Maersk) — через 1 день
└ ABCD 9876543 (MSC) — через 3 дня

💚 Приближается тарификация (4–7 дней)
└ XYZW 5555555 (Maersk) — через 6 дней

[📦 По компаниям] [⚠️ Все предупреждения] [📥 Скачать xlsx]
```

**Inline-кнопки под отчётом:**
- `📦 По компаниям` — разбивка количества и сумм по каждой компании
- `⚠️ Все предупреждения` — полный список контейнеров с эскалацией
- `📥 Скачать xlsx` — генерирует и отправляет xlsx-файл в чат

**Закреп:** бот закрепляет утренний отчёт. Предыдущий закреплённый отчёт открепляется (если бот его закреплял). Для этого храним `last_pinned_message_id` per group.

**Логика предупреждений:**
- Берём все контейнеры `on_terminal`
- Для каждого считаем `days_remaining = free_days - days_on_terminal`
- `days_remaining < 0` → 🔴 тарификация уже идёт (показываем сколько дней и сумму)
- `0 ≤ days_remaining ≤ 3` → 🟡 скоро
- `4 ≤ days_remaining ≤ 7` → 💚 приближается
- `> 7` → не показываем

### 4. Вечерний итог дня (20:00)

**Формат:**
```
📋 Итоги дня — 12.04.2026

Прибыло: +5 контейнеров
Вывезено: -3 контейнера
Выручка за вывоз: $1,240.00

📈 На терминале: 47 → 49 (+2)
💰 Общий долг: $12,450 → $13,690 (+$1,240)
```

**Логика:**
- «Прибыло» — контейнеры с `registered_at` за сегодня
- «Вывезено» — контейнеры с `departure_date` за сегодня
- «Выручка за вывоз» — сумма `total` вывезенных сегодня
- Динамика «было → стало» — разница count и суммы между утренним снимком и текущим

**Снимок:** при отправке утреннего отчёта сохраняем snapshot (count, total_debt) в память (dict). Вечерний отчёт сравнивает с ним. Если снимка нет (бот перезапущен) — показываем только абсолютные цифры без динамики.

### 5. Планировщик

**Библиотека:** APScheduler 3.x (apscheduler)
- `CronTrigger(hour=6, minute=0)` — утренний отчёт
- `CronTrigger(hour=20, minute=0)` — вечерний итог
- Таймзона из `.env` (`TIMEZONE`, default `Asia/Tashkent` — уточнить у клиента)
- Запуск планировщика в `bot.py` при старте через `on_startup` хук aiogram

**Конфиг:**
```
REPORT_HOUR=6
EVENING_REPORT_HOUR=20
TIMEZONE=Asia/Tashkent
```

## Настройка клиентом

1. Создать группу в Telegram
2. Добавить бота в группу как администратора (права: отправка сообщений, закрепление)
3. Получить chat_id группы (бот может сам сообщить при /start в группе)
4. Вписать chat_id в `GROUP_IDS` в `.env`
5. У BotFather: отключить Group Privacy (`/setprivacy` → Disable) чтобы бот видел все сообщения
